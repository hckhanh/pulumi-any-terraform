## Project Overview

A monorepo of dynamically bridged Pulumi providers that wrap Terraform providers as TypeScript npm packages. ~95% of TypeScript in `packages/` is **auto-generated** by `pulumi-language-nodejs` -- never edit files with the `// *** WARNING: this file was generated` header.

## Commands

```bash
# Install dependencies
pnpm install

# Build all packages
pnpm nx run-many -t build

# Run all checks (audit + build + syncpack + prettier)
pnpm nx run-many -t check

# Check only affected projects (faster in PRs)
pnpm nx affected -t check

# Auto-fix formatting and lint issues
pnpm nx run-many -t fix

# Format code with Prettier
pnpm nx run root:prettier:write

# Fix dependency version mismatches
pnpm nx run root:syncpack:format
```

## Architecture

### Generated vs Hand-written Code

| Location                            | Type                                        | Editable?             |
| ----------------------------------- | ------------------------------------------- | --------------------- |
| `packages/*/` (`.ts` files)         | Auto-generated Pulumi provider SDKs         | No                    |
| `packages/*/scripts/postinstall.js` | Build scripts (run `tsc`)                   | Template, rarely edit |
| `tools/`                            | Hand-written Nx build plugins               | Yes                   |
| `.github/scripts/check-updates.js`  | CI automation for upstream provider updates | Yes                   |
| `docs/`                             | Fumadocs documentation site                 | Yes                   |

### Nx Plugin System

All build orchestration is through custom Nx plugins in `tools/` that extend an abstract `Plugin` base class (`tools/utils/plugin.ts`). Each plugin matches a file glob pattern and generates Nx targets:

- `tools/build.ts` -- matches `**/*/tsconfig.json`, runs `postinstall.js` to compile TypeScript
- `tools/biome.ts` -- matches `**/biome.json`, runs Biome check/fix
- `tools/linter.ts` -- matches `**/project.json`, wires up `check` and `fix` meta-targets
- `tools/prettier.ts` -- matches `**/project.json`, runs Prettier check/write

### Provider Update Flow

`check-updates.js` runs weekly in CI: fetches latest versions from Terraform registry and GitHub releases, runs `pulumi package add` in a temp project to regenerate the SDK, copies files back, creates Changesets, and commits to main.

## Code Conventions

### Formatting (enforced by Biome + Prettier)

- 2-space indentation
- No semicolons (Biome `asNeeded`)
- Single quotes
- LF line endings
- Biome formats JS/TS/JSON/CSS; Prettier formats YAML/Markdown/HTML
- Generated `packages/` files are excluded from both formatters (`.prettierignore`)

### TypeScript (hand-written code in `tools/`)

- Target ES2022, strict mode
- Use `import type` for type-only imports
- Use `node:` protocol for Node.js builtins
- Extend the `Plugin` abstract class for new Nx plugins

### Dependencies

- All versions pinned to exact (no ranges), enforced by syncpack
- pnpm overrides in `pnpm-workspace.yaml` for security patches on transitive deps
- `allowBuilds` in `pnpm-workspace.yaml` whitelists packages allowed to run install scripts
- `minimumReleaseAge: 2880` (48h) in pnpm matches Aikido Safe Chain's default

### Commits

Conventional commits: `type(scope): description`. Common patterns:

- `chore(release): release packages`
- `chore(release): update packages to match upstream terraform providers`
- `chore(deps): update dependency X to vY`

### CI Workflows

- **test.yml** -- lint + build on PRs and main pushes
- **autofix.yml** -- auto-fix formatting, runs `autofix-ci/action`
- **publish.yml** -- build + publish via Changesets
- **update.yml** -- weekly upstream provider version check, commits directly to main
- All GitHub Actions pinned by SHA, not tags
- Aikido Safe Chain installed in all workflows for supply chain protection
